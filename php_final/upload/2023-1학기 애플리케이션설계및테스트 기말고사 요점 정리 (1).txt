
S/W 개발 기본 절차와 각 단계의 목적은? 
	요구 분석 --> 설계 --> 구현 --> 테스트
'요구 분석' 단계에서 하는 일은?
	개발할 시스템에 대한 이해, 사용자 요구(유저 니즈) 파악

요구분석을 왜 하나?  내가 만들 소프트웨어가 어떤 모습인지 정하는 것

설계를 왜 하나? SW의 기능을 구성할려고

구현 : sw를 직접 언어를 선택해서 만드는 것

테스트 : 배포전에 프로그램에 문제가 있는지 확인

개체 관계도(ERD, E-R 다이어그램)를 그리는 목적은? (entity relationship diagram) DB설계를 할려고

'논리 ERD'와 '물리 ERD'의 차이점은? 물리 ERD는 컬럼명, 데이터타입과 형식, 길이를 지정, 논리 ERD는 개체, 관계, 속성을 지정

'프로그램 처리 로직' 작성법 3가지는? 모듈명세서, N-S차트, HIPO(IPO) 차트

온프레미스(on-premises)와 클라우드(cloud)의 차이점은? 

클라우드 : 누군가가 인터넷상에서 서버를 구축해서 나는 일부분을 빌려서 사용
온프레미스 : 서버를 내가 다 구축을 하는 것 (온프레미스 영단어 쓰고 어떤 뜻인가?)

ERP가 무엇인가? 기업 업무용 애플리케이션(전사적 자원 관리) (엑셀 x, 기업 업무를 위한 소프트웨어)

ERP를 만들어서 쓰는 법(3가지) : 개발자를 직접고용, 외주 개발(Outsourcing), 돈 주고 사기

1차원 바코드와 2차원 바코드의 장단점은? 일반적으로 작대기가 1차원 qr코드 처럼 생긴게 2차원

각각의 장 단점 : 1차원 보통은 길어봤자 13자 제일긴건 128자 2차원은 1000글자가 넘어서 많은 정보를 담을 수 있는게 장점 1차원은 훼손이 되어도 읽을 수는 있음 2차원은 불가
1차원은 바코드가 완전히 훼손 되어도 글자가 있어서 입력가능 2차원은 글자가 너무 길어서 글자가 없다.

학번, 사번, ... 등등의 코드를 설계할 때 고려할 사항은? 대상수, 대상선정, 부여방식, 대상의 특성, 설계 내용 문서화

체크 디지트(check digit)가 무엇이며 사용하는 목적은? 일반 코드에다가 한 글자의 코드를 넣는 것 왜? 번호가 맞는지 확인하기 위해서 

library와 framework(영어로)가 무엇이며 차이점은? 남이 만들어 놓은 걸 가져와서 쓰는 것, 프로그램을 짜기 위한 뼈대 

API(Application Program Interface)가 무엇? API는 프로그램에서 일종의 약속, 형태 , 프로그램과 프로그램 사이에 서로 데이터를 주고 받기 위해서 주로 내 프로그램에서 다른 프로그램에게 실행을 요청
library는 쓰기 위해서 안에 뭐가 들어있는지 알아야 한다.
라이브러리랑 api는 연관이 있다.

많이 사용하는 IDE의 이름은? 자바 : 이클립스, 인텔리제이, 안드로이드 : 안드로이드 스튜디오, visual studio

S/W 개발 도구에서 plug-in 또는 add-on 또는 extention의 의미는? 추가적인 기능을 설치

	Apache (Web server), IIS, Tomcat, NGINX, GWS(Google Web Server), ...

'아스키 코드'의 용도와 구성은? 컴퓨터에 들어 가는 데이터(3가지) 수치데이터, 문자데이터, 이진데이터
	문자 데이터 저장 형식, '아스키 코드' = 영문자 + 숫자 + 특수문자 + 제어문자  
            제일 앞줄에 들어 가는 거는 제어문자
다음 용어의 뜻은? (영어로 적으시오)
	빌드(build) : 컴파일과 링크를 합친 것
	패키징(packaging) : 라이브러리를 하나로 묶는 것
	배포(deploy) : 패키징 한걸 주는 거
	출시(release) : 발표했다고 알려 주는 것
	이관(migration) - 데이터 이관(data migration) : 앞에 과정과 상관없이 하나의 컴퓨터를 다른 컴퓨터로 옮긴다.(마이그레이션)

컴파일(compile)이란? 소스코드를 기계어로 번역해주는 과정 

디버깅(debugging)이란? 오류 찾는 일
